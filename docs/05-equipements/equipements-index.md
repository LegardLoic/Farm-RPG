# Index des équipements MVP

> **Statut : document canonique actif**
> Ce document définit les équipements canoniques du MVP.
> Il sert de référence active pour le front, l’inventaire, l’équipement du joueur, le shop du Forgeron, la progression combat, les récompenses et les futurs documents détaillés sur les équipements.
> En cas de divergence avec les anciens documents de cadrage ou les archives, ce document prévaut pour les équipements du MVP.

---

## 1. Rôle du document

Ce document fixe le cadre du système d’équipement du MVP.

Il permet de :
- définir quels slots d’équipement existent réellement ;
- structurer les grandes familles d’équipement ;
- relier les équipements au système de statistiques et de compétences ;
- guider la production du front et du shop du Forgeron ;
- éviter la dérive vers trop d’objets ou trop de variantes trop tôt.

Le MVP n’a pas besoin d’un gigantesque catalogue d’équipements.
Il a besoin d’un **noyau d’objets lisibles, progressifs et suffisamment marquants pour faire sentir la montée en puissance du joueur**.

---

## 2. Principes de conception des équipements MVP

Les équipements du MVP doivent respecter les règles suivantes :

### 2.1 L’équipement change concrètement le jeu
Un équipement ne doit pas être seulement un petit bonus invisible.
Il doit idéalement :
- améliorer les stats du joueur ;
- ouvrir une option de build ;
- soutenir un style de jeu ;
- ou donner accès à une compétence liée.

### 2.2 Lisibilité avant accumulation
Le joueur doit comprendre rapidement :
- dans quel slot l’objet se porte ;
- ce qu’il améliore ;
- si l’objet soutient plutôt un style physique, défensif, magique ou utilitaire.

### 2.3 Progression liée au monde
L’équipement doit donner l’impression de progresser avec le monde :
- accès limité au début ;
- ouverture du Forgeron et de nouveaux paliers ;
- meilleure qualité matérielle après les seuils majeurs.

### 2.4 Compatibilité avec le système de build
Le MVP doit déjà soutenir l’idée centrale du projet :
- le héros commence neutre ;
- ses choix de stats et d’équipement orientent son style ;
- certains équipements peuvent porter des compétences ou exigences de stats.

---

## 3. Slots canoniques du MVP

Le système d’équipement du MVP repose sur les slots suivants :

- Casque
- Amulette
- Torse
- Jambes
- Bottes
- Gants
- Bague 1
- Bague 2
- Main gauche
- Main droite

Ces slots sont déjà cohérents avec le cadrage produit et doivent être considérés comme canoniques pour le MVP.

---

## 4. Vue d’ensemble du système d’équipement MVP

Les équipements du MVP sont répartis en **4 grandes familles** :

### 4.1 Armes
Objets offensifs principaux, surtout liés aux mains.

### 4.2 Armures
Objets défensifs ou structurels :
- casque
- torse
- jambes
- bottes
- gants

### 4.3 Accessoires
Objets de soutien ou d’orientation légère :
- amulettes
- bagues

### 4.4 Équipements à compétence liée
Certains équipements peuvent porter une compétence utilisable tant qu’ils sont équipés, puis potentiellement apprise via le système de progression prévu par le projet.

Cette dernière catégorie n’est pas une famille séparée d’objet, mais une propriété importante du système.

---

## 5. Rôle des équipements dans le MVP

Les équipements doivent soutenir 4 fonctions majeures :

### 5.1 Renforcer la survie
Plus de défense, de stabilité ou de robustesse.

### 5.2 Renforcer l’offensive
Plus de dégâts, plus de précision, plus d’impact.

### 5.3 Soutenir les styles de jeu
Permettre d’orienter le héros vers :
- force ;
- dextérité ;
- intelligence ;
- endurance ;
- polyvalence mesurée.

### 5.4 Donner de la projection
Le joueur doit sentir que certains objets ne sont pas seulement bons “maintenant”, mais dessinent déjà un futur build.

---

## 6. Raretés canoniques du MVP

Les raretés globales du projet sont :

- Commun
- Rare
- Très rare
- Épique
- Légendaire

Cependant, dans le **MVP strict**, les raretés activement exploitées doivent rester principalement :

- **Commun**
- **Rare**

Les autres niveaux peuvent exister comme horizon ou ouverture, mais ne doivent pas alourdir la production initiale.

---

## 7. Progression d’équipement recommandée pour le MVP

Le MVP doit proposer une progression simple par paliers.

### Début de MVP
- équipements très simples ;
- pièces communes ;
- amélioration faible mais lisible ;
- fonction surtout pédagogique.

### Milieu de MVP
- meilleures armes et armures du Forgeron ;
- apparition de quelques objets plus spécialisés ;
- premières sensations de build réel.

### Fin de MVP
- équipements plus marqués ;
- pièces rares plus intéressantes ;
- sentiment que le héros peut déjà commencer à ressembler à un vrai profil de combat.

Le but n’est pas encore de proposer 50 builds.
Le but est de faire sentir que :
**le héros se construit à travers ses choix d’équipement.**

---

## 8. Grandes orientations de build soutenues par le MVP

Le MVP n’a pas besoin de classes verrouillées, mais les équipements doivent déjà rendre lisibles quelques orientations.

### 8.1 Orientation force
- armes plus lourdes ;
- armures plus robustes ;
- style de jeu frontal, solide, offensif physique.

### 8.2 Orientation dextérité
- équipements plus légers ;
- meilleure précision ou mobilité ;
- style plus nerveux, plus technique ou plus agile.

### 8.3 Orientation intelligence
- soutien aux MP ;
- meilleure efficacité des compétences ;
- objets plus orientés maîtrise ou magie.

### 8.4 Orientation hybride raisonnable
- pièces intermédiaires ;
- builds encore ouverts ;
- possibilité de ne pas tout figer trop tôt.

Le MVP doit déjà laisser apparaître ces directions sans les saturer de détails.

---

## 9. Exemples canoniques d’équipements MVP

Le catalogue détaillé viendra dans les documents suivants, mais le MVP doit au minimum pouvoir s’appuyer sur des objets de ce type :

### Armes
- épée de fer simple
- bouclier de garde
- arme légère de base
- arme ou catalyseur plus orienté magie / focus

### Armures
- plastron simple
- bottes renforcées
- gants de travail / de garde
- casque léger ou robuste
- jambières simples

### Accessoires
- amulette simple de focalisation
- bague de vigueur
- bague utilitaire légère

### Pièces à compétence liée
- armure lourde donnant accès à une compétence de charge ou d’impact
- objet de focus donnant accès à une technique simple
- équipement défensif ouvrant une option utilitaire

Ces exemples ne fixent pas encore le détail exact des fiches, mais ils fixent bien le territoire du MVP.

---

## 10. Place du Forgeron dans le système

Le Forgeron est le point d’entrée principal de la progression d’équipement du MVP.

Son rôle est de :
- rendre l’équipement tangible dans le monde ;
- matérialiser les paliers de déblocage ;
- proposer une montée en qualité cohérente avec la tour ;
- faire ressentir que le village retrouve une capacité concrète à soutenir le héros.

Le système d’équipement ne doit pas sembler “tombé du ciel”.
Il doit passer par :
- la progression ;
- les déblocages ;
- l’artisanat du village ;
- le retour de fonctions utiles.

---

## 11. Lien avec les statistiques du joueur

Les équipements du MVP doivent être cohérents avec les statistiques de base du projet :

- FOR
- DEX
- CON
- INT
- VIT

Ils doivent pouvoir :
- renforcer une ou plusieurs stats ;
- demander un minimum de stats pour être équipés dans certains cas ;
- orienter le joueur vers des choix.

L’idée canonique importante à préserver est la suivante :
**l’équipement ne se contente pas d’accompagner la progression, il participe activement à la construction du personnage.**

---

## 12. Lien avec les compétences liées à l’équipement

Le système MVP doit déjà reconnaître le principe suivant :

- certains équipements peuvent donner accès à une compétence tant qu’ils sont portés ;
- cette compétence peut, plus tard ou selon le système retenu, être apprise durablement ;
- cela fait de l’équipement un vecteur de spécialisation, pas seulement de statistiques.

Le MVP n’a pas besoin d’exploiter 30 cas différents de cette logique.
Mais il doit déjà la soutenir clairement dans ses équipements marquants.

---

## 13. Contraintes de cohérence

Les équipements du MVP doivent toujours respecter les règles suivantes :

- chaque équipement doit avoir une fonction lisible ;
- pas d’explosion incontrôlée du catalogue ;
- pas de pièces trop redondantes sans intérêt de build ;
- priorité aux objets qui soutiennent les choix du joueur ;
- les objets du Forgeron doivent sembler fabriqués ou crédibles dans le monde.

Le joueur doit pouvoir se dire :
- “ça me rend plus solide”,
- “ça me donne accès à une capacité”,
- “ça pousse mon build dans une direction”.

---

## 14. Contraintes front et UI

Le front doit pouvoir afficher clairement :

- le slot concerné ;
- le nom de l’équipement ;
- sa rareté ;
- ses bonus ;
- ses prérequis éventuels ;
- sa compétence liée si elle existe.

L’interface d’équipement du MVP doit rester lisible et non intimidante.

### Priorités visuelles
- distinguer clairement armes / armures / accessoires ;
- rendre les bonus compréhensibles ;
- afficher les comparaisons simplement ;
- faire sentir la montée de qualité sans surcharger l’écran.

## 17. Résumé exécutif

Le système d’équipement du MVP repose sur un noyau clair :
- plusieurs slots canoniques,
- trois grandes familles d’équipements,
- une progression simple par paliers,
- et une logique forte de construction du héros via l’équipement.

Le MVP ne cherche pas encore à proposer un loot massif.
Il cherche à rendre l’équipement :
- lisible,
- utile,
- lié au monde,
- et déjà porteur de vrais choix de build.

Son rôle central est simple :
**le joueur ne devient pas plus fort seulement parce qu’il gagne des niveaux ; il devient plus fort parce qu’il transforme ses victoires en moyens concrets d’agir autrement.**
