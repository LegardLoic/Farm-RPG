# Index PNJ MVP

> **Statut : document canonique actif**
> Ce document recense les PNJ canoniques du MVP et fixe leur rôle narratif, leur fonction gameplay, leur état initial et leur potentiel d’évolution.
> Il sert de référence active pour le front, les dialogues, les quêtes, les interactions et la production visuelle.
> En cas de divergence avec les anciens documents de cadrage ou les archives, ce document prévaut pour le casting PNJ du MVP.

---

## 1. Rôle du document

Ce document sert d’index de référence pour les PNJ du MVP.

Il permet de :
- définir le noyau humain du village ;
- préciser qui existe réellement dans le MVP ;
- indiquer le rôle de chaque PNJ dans la boucle ferme ↔ village ↔ tour ;
- guider les futurs documents détaillés PNJ ;
- fournir une base exploitable pour les dialogues, les états visuels, les interactions front et les quêtes.

Le MVP ne cherche pas à proposer un village immense ni un grand nombre de personnages.
Il cherche à proposer un **petit groupe de PNJ identifiables, utiles et incarnés**.

---

## 2. Principes de casting du MVP

Le casting du MVP doit respecter les règles suivantes :

### 2.1 Petit nombre, forte lisibilité
Le village MVP doit contenir peu de PNJ importants, mais chacun doit être mémorisable et utile.

### 2.2 Double fonction
Chaque PNJ principal doit avoir :
- une **fonction gameplay** ;
- une **fonction narrative**.

### 2.3 Evolution visible
Chaque PNJ majeur doit pouvoir changer au fil de la progression :
- état ;
- dialogue ;
- disponibilité ;
- service ;
- relation au héros.

### 2.4 Production raisonnable
Le nombre de PNJ du MVP doit rester compatible avec :
- la production front ;
- la production portrait/sprite ;
- la production de dialogues ;
- la clarté de la boucle de jeu.

---

## 3. Casting canonique minimum du MVP

Le casting MVP repose sur **5 PNJ de référence**.

### 3.1 Le Maire
Rôle global :
- point d’entrée narratif du village ;
- donneur de la quête principale ;
- premier relais du joueur dans la compréhension de la situation.

Fonctions gameplay :
- introduction ;
- progression narrative ;
- suivi de l’avancement principal ;
- appui au système de relation simple.

Fonctions fictionnelles :
- figure d’autorité fatiguée mais encore debout ;
- incarnation du village qui essaie de tenir malgré tout ;
- premier visage humain du problème.

Etat dans le MVP :
- disponible dès le début ;
- central dans l’introduction ;
- présent ensuite comme guide et témoin de l’évolution du monde.

Priorité de production :
- **très haute**

---

### 3.2 Le Forgeron
Rôle global :
- premier grand PNJ “à libérer” partiellement ;
- incarnation concrète de la progression du joueur sur le monde.

Fonctions gameplay :
- shop d’équipement / armes ;
- progression d’offres par paliers ;
- symbole d’un premier vrai déblocage systémique.

Fonctions fictionnelles :
- artisan important du village, mais fortement affecté par la malédiction ;
- son état doit faire ressentir que certains habitants ne sont plus pleinement capables d’assurer leur rôle ;
- son retour progressif doit être perçu comme une victoire réelle.

Etat dans le MVP :
- présent, mais diminué ou partiellement bloqué au départ ;
- évolue au fil des paliers ;
- son redéploiement constitue l’un des retours monde majeurs du MVP.

Priorité de production :
- **très haute**

---

### 3.3 La Marchande du village
Rôle global :
- point d’entrée économique du quotidien ;
- interface humaine entre la ferme et le village.

Fonctions gameplay :
- vente de graines ;
- rachat des récoltes ;
- soutien à la boucle économique courte ;
- possible support de quêtes de livraison ou d’approvisionnement.

Fonctions fictionnelles :
- personnage ancré dans le réel du village ;
- donne une sensation de continuité malgré la crise ;
- aide à rendre le village vivant, concret et utile.

Etat dans le MVP :
- accessible relativement tôt ;
- peut commencer avec une activité réduite ;
- doit devenir de plus en plus utile à mesure que le monde s’ouvre.

Priorité de production :
- **très haute**

---

### 3.4 L’Herboriste / Alchimiste
Rôle global :
- figure de savoir pratique liée aux plantes, remèdes et consommables ;
- pont naturel entre la ferme et les systèmes de soin / craft / préparation combat.

Fonctions gameplay :
- support aux recettes ;
- potentiel déblocage de consommables plus avancés ;
- extension naturelle du crafting MVP ou de son immédiat après.

Fonctions fictionnelles :
- personnage lié à la connaissance du vivant, donc directement opposé à la stérilité imposée par la malédiction ;
- permet de donner une profondeur supplémentaire au village ;
- peut servir à faire sentir que certaines expertises ont été mises en sommeil.

Etat dans le MVP :
- peut être présente de manière limitée ;
- peut rester secondaire dans le MVP strict ;
- doit au minimum exister comme figure canonique déjà intégrée au monde, même si son rôle reste encore contenu.

Priorité de production :
- **moyenne**

---

### 3.5 Un Habitante ou Habitant secondaire du village
Rôle global :
- donner une densité humaine au lieu ;
- porter une ou plusieurs quêtes secondaires plus intimes ;
- ouvrir la porte au relationnel léger du MVP.

Fonctions gameplay :
- interaction simple ;
- quête secondaire ;
- variation de dialogue ;
- progression relationnelle basique.

Fonctions fictionnelles :
- représenter la vie quotidienne touchée par la malédiction ;
- montrer que le problème ne concerne pas seulement les “PNJ utilitaires” ;
- apporter de la chaleur humaine, du détail et de l’attachement.

Etat dans le MVP :
- disponible ou semi-disponible tôt ;
- peut évoluer surtout via les interactions et les petits changements du village ;
- sert à ancrer émotionnellement le lieu.

Priorité de production :
- **haute**

## 5. Etats PNJ attendus dans le MVP

Les PNJ du MVP ne doivent pas être figés.
Ils doivent pouvoir exister sous différents états.

Types d’états à prévoir selon les cas :
- absent ;
- disponible ;
- indisponible temporairement ;
- affecté par la malédiction ;
- partiellement rétabli ;
- débloqué ;
- amélioré par progression ;
- relation faible / moyenne / forte.

Le front n’a pas besoin d’un système ultra complexe de simulation sociale.
En revanche, il doit pouvoir rendre lisible :
- qu’un PNJ existe ;
- dans quel état il se trouve ;
- ce qu’il propose ;
- ce qui a changé.

---

## 6. Règles de cohérence pour les futures fiches PNJ

Chaque fiche PNJ détaillée devra contenir au minimum :

- identité du personnage ;
- rôle narratif ;
- fonction gameplay ;
- état initial ;
- état(s) de progression ;
- ton des dialogues ;
- rapport au héros ;
- rapport à la malédiction ;
- besoins front (portrait, sprite, états visuels, UI éventuelle).

Aucun PNJ ne doit être purement décoratif dans le noyau MVP.

---

## 7. Ton relationnel des PNJ

Le village doit éviter deux extrêmes :
- tout le monde totalement désespéré ;
- tout le monde exagérément léger malgré la situation.

Le ton à viser :
- fatigue ;
- retenue ;
- humanité ;
- petites différences de caractère ;
- gratitude progressive ;
- bizarrerie légère par endroits, mais jamais au point de casser l’enjeu.

Les dialogues doivent aider à sentir que :
- le village souffre ;
- le joueur dérange cette inertie ;
- les habitants recommencent doucement à croire à une amélioration.

---

## 8. Rôle du casting dans le front

Le casting MVP doit directement nourrir la transformation du front.

Cela implique que les PNJ devront progressivement être incarnés via :
- panneaux dédiés plus lisibles ;
- portraits ;
- sprites ou représentations en scène ;
- états de disponibilité ;
- dialogues contextuels ;
- interactions mieux ancrées dans le lieu.

Le but n’est plus seulement d’avoir “un bouton PNJ dans le HUD”, mais de faire sentir :
- qui parle ;
- où il se trouve ;
- pourquoi il compte ;
- ce qu’il apporte au monde et au joueur.

Ordre recommandé pour les futures fiches détaillées :

Ce choix suit la logique du MVP :

---

## 10. Résumé exécutif

Le casting canonique du MVP repose sur un petit noyau de PNJ forts :
- un maire ;
- un forgeron ;
- une marchande ;
- une herboriste/alchimiste ;
- un personnage secondaire plus humain et relationnel.

Ce choix permet de garder une production raisonnable tout en donnant au village une vraie identité.

Chaque PNJ du MVP doit avoir :
- une utilité concrète ;
- une identité claire ;
- un lien avec la malédiction ;
- une évolution visible ;
- un intérêt direct pour le front et l’expérience joueur.

Le village MVP ne doit pas être rempli de personnages.
Il doit être porté par **quelques figures marquantes, utiles et cohérentes**.
